JIRA Board & Dashboard Design

Built a complete Agile project structure from scratch: custom workflows, sprint planning, and a real-time dashboard tracking 3 parallel AI initiatives.

What I Built

Scrum Board

Custom workflow with 5 statuses: To Do, In Progress, In Review, Blocked, Done

Real-Time Dashboard

9 gadgets tracking sprint health, velocity, blockers, and cross-initiative progress

Custom Fields

Initiative tracking, Risk Level, Business Unit, and Blocker Reason fields

Sprint Structure

4 sprints with realistic historical data showing velocity progression

Context

The Scenario

I designed this JIRA setup to simulate managing 3 parallel AI initiatives for an organization looking to improve operational efficiency. The goal: give stakeholders self-service access to progress without weekly status meetings.

AI-Powered Staff Chatbot

Conversational AI for HR and IT queries

Document Classification

Auto-categorize and route documents using ML

Meeting Summarizer

Generate notes and action items from recordings

Structure

How I Organized It

Epic → Story → Task Hierarchy

3 Epics (one per initiative), 15+ Stories with acceptance criteria, Tasks for technical work, and Bugs for defects

Custom Workflow

To Do → In Progress → In Review → Done, with Blocked status for stuck items. Transitions allow moving back when needed.

Sprint Cadence

1-week sprints. Sprint 1-2 completed (historical data), Sprint 3 active (current work), Sprint 4 planned (future)

AI Initiative Delivery Tracker
